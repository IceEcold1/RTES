Rtai 4.1 and Lubuntu 14.04 LTS 32 bits installation guide

Abstract

Rtai has become one of the most used interfaces for real time applications. One of its principal advantages is its flexibility and open source orientation. Nevertheless, its installation process on Linux distributions might be quite difficult because knowledge of Linux commands and PC architecture are required. In this guide the full installation process of Rtai 4.1 on Lubuntu 14.04 LTS (32 bits) with kernel 3.10.32 (i386) is presented. Additionally, Scilab/Xcos 5.5.0, Hart tollbox 0.4.6 and Qrtailab 0.1.13 are installed for a complete real time code generation tool-chain. Special comments are also available to help you during the process.

Before you start
Create a bootable USB pen drive with the .iso image of Lubuntu 14.04 LTS for the i386 architecture (32 bits). This step is useful for newer machines without CD/DVD room. The .iso image can be downloaded from this link.

Follow these instructions:
  1. Using another machine with Ubuntu/Lubuntu, start System Tools -> Startup Disk Creator. If you use Windows, follow these instructions.
  2. In Source disc image (.iso) or CD: select the Lubuntu 14.04 image.
  3. Attach the USB pen drive and select it in Disk to use
  4. Click in Make Startup Disk and follow the instructions.
  5. Reboot the machine and change the boot order to boot from the USB pen drive.
  6. When Lubuntu ask you whether to install or test the system, select install.
  7. Follow the instructions carefully.
IMPORTANT: Update the system with System Tools>Software Updater. Once the update has finished, reboot and continue. Decline when ask to upgrade to Lubuntu 16.04.

1. Download packages and dependences

The next instructions are supposed to be implemented once Lubuntu 14.04 LTS has been successfully installed and updated. Instructions in green can be copied using Ctrl+C in this document and pasted in the terminal using Ctrl+Shift+V. An Internet connection is strictly mandatory.

1.1 Downloading and uncompressing

1.1.1 Open a new terminal with Ctrl+Alt+T. Install curl typing:

  • sudo su
  • apt-get install curl

1.1.2 Download Rtai 4.1 and the Linux kernel 3.10.32 image with:

  • cd /usr/src
  • curl -L https://www.rtai.org/userfiles/downloads/RTAI/rtai-4.1.tar.bz2 | tar xj
  • curl -L https://www.kernel.org/pub/linux/kernel/v3.x/linux-3.10.32.tar.xz | tar xJ
  • curl -L http://kernel.ubuntu.com/~kernel-ppa/mainline/v3.10.32-saucy/linux-image-3.10.32-031032-generic_3.10.32-031032.201402221635_i386.deb -o linux-image-3.10.32-generic-i386.deb

1.1.3 Uncompressing:

  • dpkg-deb -x linux-image-3.10.32-generic-i386.deb linux-image-3.10.32-generic-i386

1.1.4 Make a symbolic link to Linux and Rtai files:

  • ln -s linux-3.10.32 linux
  • ln -s rtai-4.1 rtai

1.2 Installing dependencies

1.2.1 Update apt-get database

  • apt-get update

1.2.2 Install the compiler (Build essentials)

  • apt-get install cvs subversion build-essential git-core g++-multilib gcc-multilib

1.2.3 Install Rtai dependencies

  • apt-get install libtool automake libncurses5-dev kernel-package

1.2.4 install Scilab 5.5.0 and Hart toolbox dependencies

  • apt-get install default-jre docbook-xsl fop javahelp2 libavalon-framework-java libbatik-java libfftw3-3 libflexdock-java libgfortran3 libhdf5-7 libjeuclid-core-java libjgoodies-looks-java libjgraphx-java libjhdf5-java libjlatexmath-fop-java libjlatexmath-java libjogl-java libjrosetta-java liblapack3gf libncurses5 libpcre3 libpvm3 libquadmath0 libsaxon-java libskinlf-java libstdc++6 libtinfo5 libxml2 tcl8.5 tk8.5 zlib1g libgcc1 libc6 libblas-dev libblas3gf libatlas3gf-base gfortran liblapack-dev portaudio19-dev portaudio19-doc

1.2.5 install Qrtailab dependencies

  • apt-get install libqt4-dev libqwt5-qt4-dev

2. Configuring Linux

2.1 Copy the original configuration of Linux kernel 3.10.32 and path

  • cp /usr/src/linux-image-3.10.32-generic-i386/boot/config-3.10.32-031032-generic /usr/src/linux/.config
  • cd /usr/src/linux
  • patch -p1 < /usr/src/rtai/base/arch/x86/patches/hal-linux-3.10.32-x86-5.patch

2.2 Configuring the kernel

  • make menuconfig

2.3 After doing make menuconfig a configuration window will be displayed. This step aims to achieve better performance and better real time efficiency of the Linux kernel once compiled. Because of that, be careful with the options you select. Many problems reported are related to this.

Configuration menu looks like this:

Under “Processor type and features” > “Processor family” > Select you processor family. If you have an Intel i3, i5 or i7, please chose “Core 2/newer Xeon”. For better portability, chose “Pentium Pro” for i384 (32 bit) architectures or “Generic x86_64” (amd64) for 64 bit architectures.  



Under “Processor type and features” > “Maximum number of CPU’s” > Select the number of  real CPUs (cores) of the computer. Don't take into account of cores generated by hyperthreading.   

Under “Processor type and features” > “SMT (Hyperthreading) scheduler support” > Disable


Under “Processor type and features” > Symmetric multi-processing support> Enable when your machine has more than one CPU core. 


Under “Power management and ACPI options” > Disable everything, except “ACPI”

Under “Power management and ACPI options” > “ACPI” > Disable everything, except “Power Management Timer Support” and “button”. If you are installing the system in a Notebook or other mobile device, you can can enable the "AC Adapter" and "Battery" options, although this may carry minor issues related to determinism of real time.

Under “Power management and ACPI options” > “CPU Frequency scaling” > Disable

Under “Power management and ACPI options” > “CPU idle PM support” > Disable


Once you have configured the kernel, exit and save the settings when asked.

3. Compiling and installing the Linux kernel

3.1 Compile Linux. This process might take a few hours depending on your machine, be patient.

  • make -j `getconf _NPROCESSORS_ONLN` deb-pkg LOCALVERSION=-rtai

3.2 Install packages

  • cd /usr/src
  • dpkg -i linux-image-3.10.32-rtai_3.10.32-rtai-1_i386.deb
  • dpkg -i linux-headers-3.10.32-rtai_3.10.32-rtai-1_i386.deb

4. Grub configuration

4.1 You can use grub-customizer to configure the machine to boot from Linux-Rtai. To install this application, do in the terminal:

  • sudo add-apt-repository ppa:danielrichter2007/grub-customizer
  • sudo apt-get update
  • sudo apt-get install grub-customizer

Once installed, run it. In the tab General Settings in the option default entry, select predefined and chose Advanced options for Ubuntu>Ubuntu, with Linux 3.10.32-rtai. In kernel parameters introduce “quiet splash lapic=notscdeadline”, without quotation marks. Click in Save. Wait until the new configuration is done. Close the application. Configuration should look like this:


5. Configuring Rtai 4.1

5.1 To configure the new Rtai 4.1 kernel, open the configuration menu:

  • sudo su
  • cd /usr/src/rtai
  • make menuconfig

Configuration menu should look like this:


Adjust the Rtai and Linux paths as follows:

Under “General” > “Linux source tree” = /usr/src/linux
Under “General” > “Installation directory” = /usr/realtime


Under “General” > “Inlining mode of user-space services” > Enable “Eager inlining”


Under “Machine” -> Number of CPUs (SMP only) -> select the number of CPUs (cores) of the computer. Use the same quantity of the Linux configuration menu.  


 Under “Add-ons” > Disable “Comedi support”


Exit saving changes.

6. Compiling and installing Rtai 4.1

6.1 To compile and install  Rtai do the following:

  • sudo su
  • make -j `getconf _NPROCESSORS_ONLN`
  • make install

7. Start Up

7.1 Create a file named rtai.conf in /etc/ld.so.conf.d/ with the content  "/usr/realtime/lib" without the quotation marks. After that, update system configuration with:

  • ldconfig

7.2 Applications running in real time shall use the kernel modules provided by Rtai. To automatically load these modules during the startup of the system, modify rc.local in /etc/rc.local with the following command:

  • sudo nano /etc/rc.local

Nano is a simple and powerful text editor that runs in the terminal. Nano should display the content of rc.local like this:


Add the following lines just before exit 0. Lines preceded by # will be ignored.

#Load Rtai modules

/sbin/insmod /usr/realtime/modules/rtai_hal.ko

/sbin/insmod /usr/realtime/modules/rtai_lxrt.ko

/sbin/insmod /usr/realtime/modules/rtai_fifos.ko

/sbin/insmod /usr/realtime/modules/rtai_sem.ko

/sbin/insmod /usr/realtime/modules/rtai_mbx.ko

/sbin/insmod /usr/realtime/modules/rtai_msg.ko

/sbin/insmod /usr/realtime/modules/rtai_netrpc.ko

/sbin/insmod /usr/realtime/modules/rtai_shm.ko


The file rc.local should look like this:



Do Ctrl+x to close and then Y and <enter> to save the changes and exit.

Reboot the system and do:

  • lsmod

The command lsmod will allow you to check if modules were successfully loaded.


8. First real time test (optional)

8.1 You can make a latency test to verify the right behavior and real time capabilities of the system. It’s a god idea to run these tests during stress situation, like Internet navigation, video streaming or gaming. In an ideal Rtai environment, latencies should keep at minimum and overruns should stand in zero independently of CPU’s load.

Kernel space test:

  • sudo su
  • cd /usr/realtime/testsuite/kern/latency
  • ./run

User space test:

  • cd /usr/realtime/testsuite/user/latency
  • ./run

9. Path setting

Warning: Before continuing the installation process, make sure the system recognizes the Rtai configuration file. To do so, do:

  • rtai-config --version

The system should respond with the version number of Rtai, in this case 4.1. If the verification fails (the system returns “The program ‘rtai-config' is currently not installed”), do the following:

  • cd /usr/src/rtai
  • sudo sed -i 's/\(PATH=\"\)/\1\/usr\/realtime\/bin:/' /etc/environment

Then run the following commands in every opened terminal:

  • export PATH=/usr/realtime/bin:$PATH
  • source ~/.bashrc

Reboot and verify with rtai-config –version again.


10. Installing the code generation software

10.1 Installing Scilab/Xcos 5.5.0:

  • sudo apt-get install scilab

10.2 Installing QRTAILab 0.1.13. Note: before you start installing QRTAILab you should already have loaded the Rtai modules.

  • sudo su
  • cd /usr/local/src
  • svn checkout http://svn.code.sf.net/p/qrtailab/code/trunk qrtailab
  • cd qrtailab
  • qmake-qt4
  • make
  • make install
  • exit

10.3 Installing Hart Toolbox

  • sudo su
  • cd /usr/
  • curl -L http://sourceforge.net/projects/hart/files/HART%20Toolbox/hart_0.4.6/hart_0.4.6.tar.gz | tar xz
  • cd /usr/share/scilab/contrib
  • ln -s /usr/hart_0.4.6
  • cd hart_0.4.6

Warning: DO NOT do sudo make, or Rtai module will not be generated. Instead, do sudo su first and then make.

  • make
  • exit

Reboot the system.

Check if the Hard Toolbox has been installed properly by going to Scilab and then to the 'Toolbox' option, in which 'hart_0.4.6' should appear.

Rtai and the full tool-chain must be installed in this moment.
